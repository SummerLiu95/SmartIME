# Execution Plans

Use this directory to record implementation details after a task plan has been confirmed and executed.

Do not use this directory for task planning. `docs/TASKS.md` is updated only in two cases:

- the user explicitly asks AI to plan tasks
- work is being planned in Plan mode

Create a record here only after one of those planned tasks is confirmed for execution and the implementation work has been completed.

Recommended file naming:

```text
YYYY-MM-DD-feature-or-fix-name.md
```

Each completion record should include:

- context and goal
- relevant requirements or design links
- implementation summary
- affected files and modules
- validation performed
- follow-up notes or known risks
